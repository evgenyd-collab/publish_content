# Source Code Rules

Source Code (src) - основная директория исходного кода React-приложения.
Содержит компоненты, утилиты, хелперы и стили приложения.

# Structure
Основная директория для работы: `src`.
`App.jsx` - корневой компонент приложения.
    `App` - главный компонент с роутингом и управлением темой.
        Инициализирует тему из localStorage при монтировании.
        Управляет переключением между светлой и темной темой.
        Настраивает роутинг через React Router.
        Условно отображает переключатель темы.
`BonusList.jsx` - основной компонент для работы со списком бонусов.
    `BonusList` - компонент списка бонусов с фильтрацией и пагинацией.
        Управляет состоянием загрузки бонусов и букмекеров.
        Реализует фильтрацию по статусам, флагам и доменам.
        Обрабатывает пагинацию и сортировку.
        Интегрируется с API для получения данных.
`main.jsx` - точка входа приложения.
`index.css` - глобальные стили приложения.
`App.css` - стили для основного компонента.

## Components Directory
`components/` - компоненты организованы по атомарному дизайну:
- `atoms/` - базовые неделимые компоненты
- `molecules/` - составные компоненты из атомов
- `modals/` - модальные окна

## Helpers Directory
`helpers/` - вспомогательные функции и константы:
- `constants.js` - константы приложения (локали, статусы, поля)
- `bonus-convert.js` - конвертация данных бонусов
- `terms-convert.js` - конвертация условий бонусов
- `getStatusBadge.jsx` - компонент для отображения статусов
- `redirect-if-server-page.js` - логика редиректов

## Utils Directory
`utils/` - утилиты для форматирования и обработки данных:
- `formatters.js` - функции форматирования (даты, возраст бонусов)

## Assets Directory
`assets/` - статические ресурсы (изображения, иконки).

# Component Architecture

## Atomic Design Pattern
Компоненты организованы по принципу атомарного дизайна:

### Atoms (Атомы)
Базовые неделимые компоненты:
- `add-bonus-button` - кнопка добавления бонуса
- `flags-indicator` - индикатор флагов
- `icon-button` - кнопка с иконкой
- `spinner` - индикатор загрузки
- `table-head` - заголовок таблицы

### Molecules (Молекулы)
Составные компоненты из атомов:
- `bonus-details-page` - страница деталей бонуса
- `bonus-item` - элемент списка бонусов
- `pagination` - компонент пагинации

### Modals (Модальные окна)
Специализированные компоненты для взаимодействия:
- `add-terms-modal.jsx` - модал добавления условий
- `details-modal.jsx` - модал деталей
- `manual-flags-edit-modal.jsx` - модал редактирования флагов
- `new-bonus-modal.jsx` - модал создания бонуса
- `status-edit-modal.jsx` - модал редактирования статуса

# Data Management

## State Management
- Локальное состояние через React hooks
- Подъем состояния для общих данных
- Контекст для глобальных настроек (тема)

## API Integration
- Централизованные API вызовы в основных компонентах
- Обработка состояний загрузки и ошибок
- Кэширование данных в состоянии компонентов

## Data Flow
1. API запросы в родительских компонентах
2. Передача данных через props
3. Локальная обработка и фильтрация
4. Обновление состояния через callbacks

# Styling Guidelines

## CSS Organization
- Глобальные стили в `index.css`
- Компонентные стили в `App.css`
- Inline стили для динамических значений
- CSS классы для темизации

## Theme System
- Светлая и темная темы
- CSS классы для переключения
- localStorage для сохранения предпочтений
- Динамическое применение к body элементу

# Development Guidelines

## Component Creation
- Функциональные компоненты с hooks
- Деструктуризация props
- Мемоизация для оптимизации
- Правильная обработка зависимостей useEffect

## Code Organization
- Импорты в начале файла
- Константы перед компонентом
- Хуки в начале компонента
- Обработчики событий после хуков
- JSX в конце
